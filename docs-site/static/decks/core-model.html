The Core Model

Five nouns. Name them and the rest is detail.

Gourav Shah  ·  Initcron AI / OpsFlow LLC / School of DevOps

01

Pack — the distribution unit

A directory with an AOH.yaml manifest and at least one skill.

Minimum viable pack: AOH.yaml plus one skill — that's a complete, installable unit.

02

Skill — the capability

SKILL.md plus optional scripts, references, and assets — the agentskills format.

A process skill is just a plain skill whose body orchestrates other skills by name — no separate "workflow" kind.

03

Role — the WHO

A job function that bundles the skills, model profile, and runtime requirements it needs.

Examples: sre-platform, devops-automation, mlops-training — a job function, not a person.

04

Binding — the WHERE

A role bound to a target: cluster, environment, or account. Lives outside the pack.

Same split Ansible draws between reusable roles and private inventory — the pack stays portable.

05

Adapter — compiles pack + binding to runtime

All runtime-specific knowledge lives here — never in the pack spec itself.

Hermes, Claude Code, and Codex adapters are shipped today; Goose is next on the roadmap.

06

How it composes

A team groups roles; a role pulls together everything it needs.

Bind a role to a target, run it through an adapter, get a generated, disposable runtime profile.

07

Progressive disclosure — four layers

Only the first two are mandatory. The org model is opt-in.

Minimum viable pack: AOH.yaml + one skill. Five minutes to value — the org model is something you grow into.

08

FIVE NOUNS, ONE MODEL

Pack · Skill · Role · Binding · Adapter

Name them, and the rest of the spec is just detail.
